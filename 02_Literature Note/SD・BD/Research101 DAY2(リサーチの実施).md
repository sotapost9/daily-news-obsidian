- **インタビューではイントロも工夫し、相手を立ち止まらせ、会話モードにさせる必要がある**
	- **どのくらい時間がかかるか**
	- **トーンやテンション**
	- **自分たちが興味深い存在であるアピール**
		- **Ex：「私たちデザインチームなんですけど」→新橋で立ち止まる**
	- **話した内容を何に活用するか、トピックの具体説明、質問の数を具体で決めて時間想定を伝える、インセンティブがあればそれも伝える**
	- **セッション中に何を被験者から得たいかも明確にする**
	- **セッションの構成と形式を説明し、これから何を行うかも知らせる**
	- **被験者や被験者のプロトタイプを使う能力を批判している感じが出ないように「質問に正しい答えも間違った答えもありませんが」「何か難しいと感じることがあれば、他の人もそのはずなので参考になります」とかも添えてあげる**
	- ユーザーインタビューだったら、「自分たちは第３者なので、使いづらくてもいってください。傷つきません」ということも大事。実際自分が作っていても
	- インタビュイーは企業からのインタビューを受けることは初めての場合、かなり緊張している可能性がある。
- リサーチ倫理
	- 善を行うことと、害を及ぼさないことがリサーチでは意識すべき。
	- **Do no harmは難しいが、目安としてはインタビュイーが「あれ言わなきゃ良かったな」というところをちょっと後悔するくらいがギリギリの塩梅**
	- **相手が感情的に苦しんでいるところがあれば、お水飲んでください、休憩とりますかみたいなことを聞いてあげたほうがいい**
	- **インタビューでは意外な相手の人生観に関与することがある。**
		- **携帯のインタビューで、元彼との思い出に紐づくこともある**
		- **チーム内であらかじめ想定しておくことも大事**
- インタビューの流れ
	- 通常はイントロ、ボディ、クロージングが基本だが、コツとしては自然な会話の流れのようになるべき
	- イントロ
		- 自己紹介、チーム紹介
		- インタビュー目的
		- 同意書
	- **ボディ**
		- **ワームアップ**
			- 最初の数問は、シンプルに答えられるものを。
			- 目標としては、快適な環境をセットアップする（儀式的だな）
		- **ハイレベルから始め、**
			- **時間経過に伴**う一連の出来事を自分の言葉で説明させるためのハイレベルな探りを入れる
			- 時間経過をたどって、薄く広くまず聞くことが大事なんだな。その中でディープダイブすべきことが出てくるはず。
		- **その後ディープだいぶしたいものを深ぼっていく**
			- 逆に言えば、ハイレベルの段階で後続の深い質問との関連性を意識したほうがいいのかもな。チームの関連性、デジタルーアナログ度、1週間の流れなど
			- ここでは複雑にしすぎるよりも、気になる部分についてオープンクエスチョンをばっこり投げるほうが良さそう（ーについて教えてください・見せてください）
	- クロージング
		- 参加者にハイレベルなFBを得る。
		- **参加者に優先順位をつけてもらったり、**質問があればしてもらう。
- インタビューを開始するときに心がけること
	- **意図を明確にする**
	- **親切である**
	- **参加者は専門家である**
		- **インタビュアーとインタビュイーは同等。**
	- **インタビューに集中する**
		- **目をちゃんと合わせることが大事。**
		- **メモをとり続けて、自分の頭を相手に見せるようなことはダメ。時々小さなメモを取るならいいが**
- ディープダイブするときに
	- **巻き込む、巻き込む、巻き込む**
	- **可能な限り、話すより見せる**
	- インタビューは会話ではない
	- **参加者の世界を定義させる**
		- **相手が間違った言葉の定義をしていたとしても、相手の定義に従うべき。スマホをルーターと呼んでいたとしても、それを否定せず、「ルーター」という言葉の定義を教えて、と従う姿勢を見せる**
		- **高齢者が支離滅裂な回答をしていても、可能な限り誘導しすぎないし、訂正もしない。相手に乗って、具体を聞きまくる**
- よりインタビューの質を高めるために
	- 静寂は味方
		- 沈黙は少し不快だが、それで相手が何かを伝えてくれることもある。
	- 静かに肯定する
	- 相手の言葉を反映させる
	- **理由を、攻撃的ではない方法で探ることが大事**
		- **なぜ、なぜ、なぜと繰り返すと攻撃的になってしまう**
		- **理由を知りたければ、「へー興味深いです。もっと教えていただけますか」とかで聞いたほうがいい。**
- **設問設計アクティビティの気づき**
	- **ジャーニーでは、始まりと終わりを明確にするための定義が必要**
		- **朝食体験であれば、始まりは朝起きた時間なので、そこは聞くべき。終わりの時間は仕事の時間とか**
	- **最初に「朝食」と概念の周辺にあるマインドマップや、たしからしさのある列挙をして整理をすべきだった。**
		- **朝食のタイムラインについても、家族の重要性についても、仕事や1日の時間の流れについても、影響を受けるものはたくさんあるが、ジャーニーや家族に飛びついてしまった感がある**
	- **ジャーニーの活動付箋は、その中身や質にまで深ぼる必要がある**
		- **ただ食べるだけでなく、その量や質や栄養、満足度、会話の種類なども聞いていく必要がある**
	- **話すより見せてもらうほうが多くの場合はベター。**
		- **今回なら、冷蔵庫の写真や、キッチンの写真、朝食や食卓シーンの写真も重要になってくるはず。**
	- **食べなかった時の影響もある**
		- **NULLのケースがあるのなら、その時の良し悪しも聞くべき。**
- **フィールドリサーチで意識すること**
	- **積極的に参加する**
		- **プロトコルの流れに沿って積極的に質問。詳しく知りたいことを質問する**
	- **写真を無限に撮る。あとでじゃダメ**
		- **写真は撮りまくったほうがいい。感動やストーリーもメモする**
- **インタビューTips**
	- **代名詞や、主語の欠如を避ける**
		- **シンセするときに「それ」しか残っていないとか、主語が欠けているとか、たどる面倒さが出てくる**
- **クローズドクエスチョンにも利点はある**
	- **標準化、素早く回答を聞ける、客観的に比較できる**
- **ただし、クローズドクエスチョンはアンケートでいいじゃんとなってしまうこともある。オープンクエスチョンを使うこと大事**
	- **バイアスのない、フラットな質問は回答者に負担を与えることでもあるので少し億劫だが、大事。待ちの姿勢や宙ぶらりんに耐えること大事。**
- **二重質問は避ける**
	- **あなたにとって面白く、重要ですか？みたいなのは２つの要素を正確に測れないので、分けるべき**
- **インタビュアーは自己開示しすぎないほうがいい。**
	- **特に、インタビュイーと立場が異なる場合、インタビュイーは「これを言ったも多分伝わらないんじゃないか」と発言をセーブしてしまう可能性がある。子供を持っているかどうかとか**
	- **極論、インタビュアーの意見や情報はいらない。最低限の自己開示はアイスブレイクとして機能するが。**
- **準備のTips**
	- **前日**
		- **デバイスの掃除や充電**
		- **予約や場所の確認**
	- **セッション中**
		- **時間をかけて適切かつ十分なセッティング**
		- **録音できているかの確認**
		- **光を意識：逆光じゃないか**
		- **終わった後も録音を続ける：率直なことが言えたりする**
	- **クロージング**
		- **録画できているか確認**
		- **インセンティブ支払われているか確認**
- インタビュー実践
	- **ペアの時の気づき**
		- **答えに飛びつかない**
		- **1人でインサイトは出ない**
		- **インサイトが出てきたと思ったら、何かを見過ごしている**
		- **抽象的な質問をして答えられなかったら、答えがないというだけで、それも気づき**
	- **自分の時の気づき**
		- **次の質問を考えながらは、相手の話を聞いていない証。**
			- **いうて難しいが。**
		- **自分はフィラーのタイミングはいいが、ワーディングによっては興味がなさそうに見えたり、相手に圧をかけてしまう**
			- **「ふーん」が口癖になっているらしい。相手に興味を持っていないように見える**
		- **また、沈黙が少し長いと相手が「間違ったこと言ったかな」と不安になってしまうこともある**
			- **自分はのったり淡々としているときの差が大きいからなのかな。淡々としたムードを意識すべきかも**
		- **話題の転換がうまい。次の話題に行くときに違和感なく切り替えられているので、スムーズに話せる**
