### 製品開発プロセス

- [[製品開発ライフサイクルの詳細]]

**製品開発ライフサイクルの5段階**:
1. **ブレインストーム**: ユーザーの問題をリストアップ、競合調査
2. **定義**: 誰のため？何をする？どんな機能が必要？
3. **デザイン**: ワイヤーフレーム→プロトタイプ
4. **テスト**: 社内テスト→利害関係者テスト→外部テスト
5. **ローンチ**: リリース、振り返り、次のイテレーション

**UXデザイナーの役割**: デザイン段階で最も活躍するが、ライフサイクル全体に関与

- [[デザインスプリントとは？成功例から学ぶメリットとデメリット - Miro]]

**デザインスプリントの価値**:
- 短期間（通常5日間）でアイデア→プロトタイプ→テストを完結
- チーム全体で集中的に問題解決
- 早期にユーザーフィードバックを得られる

**プロトタイピングの段階**:
- **ワイヤーフレーム**: 製品のアウトライン・スケッチ
- **ローファイプロトタイプ**: 簡易的な機能モデル
- **ハイファイプロトタイプ**: 実製品に近い詳細なモデル

## 💡 実践への応用

### 自分のプロジェクトに活かすチェックリスト

#### ブレインストーム段階
- [ ] ユーザーの問題をリストアップしたか？
- [ ] チームに多様性があるか？
- [ ] 競合製品を調査したか？
- [ ] 市場のギャップを特定したか？

#### 定義段階
- [ ] 誰のための製品か明確にしたか？（WHO）
- [ ] 製品は何をするのか明確にしたか？（WHAT）
- [ ] どんな機能が必要か明確にしたか？（HOW）
- [ ] ユーザーに直接確認したか？（仮定していないか？）

#### デザイン段階
- [ ] ワイヤーフレームを描いたか？
- [ ] プロトタイプを作成したか？
- [ ] 各機能が直感的に統合されているか？
- [ ] ユーザーにとって明確で理解しやすいか？
- [ ] アクセシビリティを考慮したか？

#### テスト段階
- [ ] 社内テストを実施したか？
- [ ] 利害関係者テストを実施したか？
- [ ] 外部テスト（実ユーザー）を実施したか？
- [ ] フィードバックを収集して改善したか？

#### ローンチ段階
- [ ] 製品をリリースしたか？
- [ ] プロモーションを開始したか？
- [ ] 振り返りを実施したか？
- [ ] 次のイテレーションの計画を立てたか？

## 📊 製品開発ライフサイクルの図解

```
ブレインストーム → 定義 → デザイン → テスト → ローンチ
     ↑                                          ↓
     └──────────────────────────────────────────┘
              （イテレーション）
```

## 🎓 このコースから得られる最も重要な学び

1. **UXデザインはユーザー中心**: 自分の仮定ではなく、ユーザーの声に基づく
2. **プロセスは反復的**: デザイン→テスト→改善のサイクルを回し続ける
3. **アクセシビリティは最初から**: 後付けではなく、設計段階から組み込む
4. **チームで働く**: UXデザイナーは単独ではなく、多職種と協働する
5. **競合は評価フェーズで**: 最初から競合を見ると、潜在ニーズを見落とす
