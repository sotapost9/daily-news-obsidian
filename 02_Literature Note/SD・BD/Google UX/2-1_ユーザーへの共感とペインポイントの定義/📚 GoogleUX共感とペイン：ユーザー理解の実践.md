### 共感マップ（Empathy Map）

- [[Empathy Mapping The First Step in Design Thinking]]
- [[Empathy Maps Put yourself into your customer's shoes]]

- [[共感マップ作ってみる]]

**共感マップの4象限**:
- **SAY（言うこと）**: インタビューでの発言
- **DO（すること）**: 実際の行動
- **THINK（考えていること）**: 内面の思考
- **FEEL（感じていること）**: 感情

**重要な洞察**: 
> 「共感マップは、各インタビューを消化しやすい情報の断片に分解するのに役立つ」

- インタビュイーごとに同じフレームで整理できる
- 漏れなく考えられる
- 発言や行動の裏を考え尽くすことができる
- ファクトと憶測を整理しやすい

### ペルソナ（Persona）

- [[Personas – A Simple Introduction]]
- [[ペルソナの作成]]

**ペルソナの要素**:
- 名前、年齢、職業などの基本属性
- 目標（Goals）
- 課題（Pain Points）
- 動機（Motivations）
- 行動パターン（Behaviors）

**注意点**: 
- デモグラフィック（属性）だけでなく、「思考様式（Thinking Styles）」で語るべき
- 属性ではなく、動機・感情・推論を中心に

### インタビュー技術

- [[インタビューにおいて注目すべきこと]]
- [[インタビューの質問文は、短く、シンプルであれ。]]
- [[参加者に詳しく説明させるよう促すことが大事]]
- [[質問は中立的に。誘導的にしたり、ストレスを与えたりするものであってはいけない]]

**インタビューの原則**:

1. **質問は短く、シンプルに**
   - 複雑な質問は避ける
   - 一度に一つのことだけを聞く

2. **中立的な質問**
   - 誘導的な質問は避ける
   - ストレスを与えない
   - 例：「〇〇は便利ですよね？」→ ❌
   - 例：「〇〇を使ったときどう感じましたか？」→ ✅

3. **詳しく説明させる**
   - オープンクエスチョンを活用
   - 「もう少し詳しく教えてください」
   - 沈黙を恐れない

4. **注目すべきポイント**
   - 言葉だけでなく、表情・声のトーン・姿勢
   - 矛盾や違和感
   - 「あれ？」と思った瞬間

関連: [[インタビュー技術の本質]]、[[観察の技術と哲学：深く見る力の育て方]]

### ペインポイント（Pain Points）

- [[4つのタイプのペイン]]
- [[ペインとソリューション]]

**4つのタイプのペイン**:

1. **機能的ペイン（Functional）**: タスク完了の障害
   - 例：アプリの読み込みが遅い、操作が複雑

2. **経済的ペイン（Financial）**: コストの問題
   - 例：価格が高い、隠れたコストがある

3. **プロセスペイン（Process）**: 手順の問題
   - 例：登録に時間がかかる、手順が多すぎる

4. **サポートペイン（Support）**: サポートの問題
   - 例：問い合わせに返信がない、ヘルプが不十分

**ペインとソリューション**:
- **ペイン**: 「〜する必要がある」「したい」「〜の必要がある」など。渇望や痛みや願い。
- **ソリューション**: 解決策や具体施策。ペインに応えるもの。
- この明確なQ&Aで捉えることが簡潔

関連: [[ジョブ理論の本質と実践]]

### リサーチの姿勢

- [[リサーチにおいては、仮説は捨てるべき。だが推論はすべき。]]

**仮説 vs 推論**:

| 仮説（Hypothesis） | 推論（Inference） |
|-------------------|------------------|
| 事前に答えを決めている | データから導き出す |
| バイアスを生む | 柔軟な解釈 |
| 検証するもの | 発見するもの |

**リサーチの姿勢**:
- 仮説を持ちすぎると、それを証明するデータだけを探してしまう（確証バイアス）
- しかし、推論（データから意味を見出すこと）は必要
- **「わからない」状態を受け入れる勇気**

## 💡 実践への応用

### インタビュー実施のチェックリスト

#### 準備段階
- [ ] インタビューの目的を明確にしたか？
- [ ] 質問リストを作成したか？（ガイドとして）
- [ ] 質問は短く、シンプルか？
- [ ] 質問は中立的か？（誘導的でないか？）
- [ ] 録音・記録の準備はできているか？

#### 実施中
- [ ] ラポール（信頼関係）を構築できているか？
- [ ] オープンクエスチョンを使っているか？
- [ ] 「もう少し詳しく」を使っているか？
- [ ] 沈黙を恐れていないか？
- [ ] 非言語コミュニケーションも観察しているか？
- [ ] 矛盾や違和感に気づいているか？

#### 事後分析
- [ ] 共感マップを作成したか？（SAY, DO, THINK, FEEL）
- [ ] SAYとDOの矛盾を特定したか？
- [ ] 4つのタイプのペインを分類したか？
- [ ] ペルソナを作成したか？
- [ ] 推論はデータに基づいているか？（仮説に引っ張られていないか？）

## 🎨 共感マップのテンプレート

```
┌─────────────────────┬─────────────────────┐
│  THINK（考える）      │  FEEL（感じる）       │
│                     │                     │
│ - 内面の思考         │ - 感情              │
│ - 懸念              │ - 気分              │
│ - 疑問              │ - 態度              │
│                     │                     │
├─────────────────────┼─────────────────────┤
│  SAY（言う）         │  DO（する）          │
│                     │                     │
│ - 発言              │ - 実際の行動         │
│ - インタビュー内容   │ - 観察した行動       │
│ - 表に出る意見      │ - 具体的なアクション  │
│                     │                     │
└─────────────────────┴─────────────────────┘
```

**重要**: SAYとDOの矛盾こそが、潜在ニーズを見つける鍵

## ✨ このセクションから得られる最も重要な学び

1. **SAYとDOの矛盾に注目**: 言っていることと実際の行動の違いが潜在ニーズ
2. **仮説は捨てて、推論はする**: バイアスを避けつつ、データから意味を見出す
3. **質問は短く、中立的に**: 誘導せず、ストレスを与えず、シンプルに
4. **共感マップで構造化**: インタビューを消化しやすい情報に分解
5. **4つのペインで分類**: 機能的・経済的・プロセス・サポート
