# 第一章：人の欲しいは隠れている

- インサイトの概要
    - 「人を動かす隠れた心理」
- インサイトの条件
    - 95%の無意識にある。論理的に考えて出るものは偽物
    - いいインサイトは刺激されて初めて気づく
        - スマートロックで「鍵開けるって面倒だったんだ」
        - 消費者は答えを持っていない
        - インサイトと違いニーズは顕在化しており、皆が自覚しており、もう既に満たされ尽くしている
        - キーインサイトとバリュープロポジションはよく使う典型パターン。インサイトと提案価値、アイデアをセットで示せる

# 第二章：人を動かす隠れた心理＝インサイトの構造を理解する

- 「**客観的な事実に基づく感情」がインサイト**
    - 単なる感情をインサイトと読んではいけない。それを引き起こす要因となっている事実をセットでインサイトを語らないと、アイデアの根拠を用意することができず、ブレる。
- **インサイトを構成する4つの要素**
    - ①シーン（場面）：感情が生まれた場面。行動や状態を伴う。Ex）日時、季節、場所、環境、一緒にいる人、何をしていたか、どんな時間だったか、どんな状態だったか、前後に何をしていたか
    - ②ドライバー（直接要因）：感情を生み出す元となった直接的な要因。Ex）物性・素材・成分、機能や性能、実施に行った行動や行為、感じたkんかく、歴史や由来やエピソード、背景にある事実
    - ③エモーション（感情）：気分や気持ち、情緒。唯一主観的で良いもの。
    - ④バックグラウンド（背景）：感情がペインやゲインである背景的な理由。Ex)出来事や経験、周囲の環境、人間関係、労働環境、社会的な影響、自身に関すること
    - 
    
    ![image.png](%E3%80%8C%E6%AC%B2%E3%81%97%E3%81%84%E3%80%8D%E3%81%AE%E6%9C%AC%E8%B3%AA%20%E3%82%A4%E3%83%B3%E3%82%B5%E3%82%A4%E3%83%88%20112f5ae744f98008a69bf88f3665f3a8/image%202.png)
    
- **インサイトは3つのタイプがある**
    - ①価値：ある対象へのポジティブな心理。購入や継続利用、好きという気持ちなど、対象への肯定的な行動をとる。
    - ②不満：ある対象へのネガティブな心理。買わない、嫌いになる、やめるなどのネガティブな行動をとる。
    - ③未充足欲求：ある対象への求めているがまだ充たされていない心理。何がどうなったらいいなと感じるか。充たされると、買いたくなる、替えたくなる、好きになる、行いたくなる。
- **キーインサイトの３条件**
    - ①バリュープロポジションと満たし満たされの関係になっていること
    - ②不満、または未充足欲求のエモーションと、他の要素の組み合わせで構成されていること
    - ③②の他の要素とは、シーン、背景、ドライバーのいずれか一つ以上であること
- **エンジェルインサイトには要注意。デビルを疑ってみる**
    - Ex：マックのユーザーにはヘルシーを求めるリサーチも多かったが、実際にはビックマックにかぶりつきたいというデビルインサイトの方が強かった。（リサーチ環境の違い、マーケティングリサーチではなくデザインリサーチの重要性にもつながるな。）
    - 人には必ず善と悪の部分が両面ある。善だけ考えていては綺麗事になってしまうので両面から考えることが必要。
    - 欲望曼荼羅：かなり大まかな分類にはなるが、多くの事例がこの中のどれかのカテゴリに属することは認めやすい。考えたいインサイトがどこにあるのかを見れば、解像度が少し上がるかも。

# 第三章：ユーザーや競合ではなく、人を見にいく

- 人間を見にいく＝ターゲットの興味や関心に寄り添うこと
    - 消費者に答えを求めても失敗する

# 第四章：成熟市場におけるビジネス機会の見つけ方

- インサイトを活かす、見つけるための手順
    - **①Opportunity発見**
        - Opportunity＝自社のビジネス（カテゴリーやブランド）で解決できそうな、消費者が潜在的に感じている課題
        - Ex：昼寝が流行っているが、隙間時間の活用をしたいのでは？
        - **手法：スペースファインダー**
            - テーマ設定→テーマに沿った新奇事象収集→事象を2分する軸を作る→４つの象限に事象をプロットする→既存商品もプロットする→新奇事象はあるのに既存商品が存在しないエリアをオポチュニティとする→これをたくさんいろんな軸で組み合わせ繰り返す。
        - **手法：Webアクチュアルデータ分析**
            - Google Trendsで気になるキーワードで過去5年分の検索ボリュームを見る→興味が高まっているならその背景の問題を考え、他のキーワードとの比較を用いて探る。
        - **手法：玉度モデル**
            - アクチュアルデータで得られたものは玉石混合なので、それを検証する
            - 魅力度×未充足度の１〜５で。
            - 魅力あるけど充足している場合、差別化できるなら攻めていい。
    - **②Insightリサーチ**
        - 潜在的な問題の解決方法を突き止めるために具体的にどのような不満や欲求なのか
        - Ex：昼寝や隙間時間活用のオポチュニティはあるが、そこに具体的な不満や欲求はどんなものが隠れているのか？
    - ③Ideation
    - ④Prototyping
