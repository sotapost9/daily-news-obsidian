# デザインリサーチで使えるジョブインタビュー実践ガイド

「ユーザーインタビューをしても、表層的なニーズしか取れない」——多くのデザイナーが抱える悩みです。顧客に「何が欲しいですか？」と聞いても、得られるのは「もっと速く」「もっと安く」という抽象的な要望ばかり。

ジョブ理論に基づくインタビュー手法は、この問題を解決します。この記事では、「短編ドキュメンタリー」のフレームワークを実際のインタビューで使う方法を、具体的な質問例とともに解説します。

## なぜ普通のインタビューでは深掘りできないのか

### 問題1: 未来の仮定を聞いている

**NG質問：** 「どんな機能があったら嬉しいですか？」

**問題点：** 人間は未来の自分の行動を予測するのが苦手です。「あったら嬉しい」と答えた機能を、実際には使わないことが多々あります。

### 問題2: 抽象的すぎる

**NG質問：** 「どんな課題がありますか？」

**問題点：** 抽象的な質問には抽象的な答えしか返ってきません。「時間がない」「使いにくい」——これではデザインに落とし込めません。

### 問題3: 機能に焦点が当たっている

**NG質問：** 「この機能について、どう思いますか？」

**問題点：** 機能の評価を聞いても、その奥にある「達成したい進歩」は見えません。

## ジョブインタビューの3つの原則

### 原則1: 過去の具体的な体験を聞く

**なぜ？** 過去の実際の行動は嘘をつきません。人は未来を予測できませんが、過去は正確に覚えています（特に最近の出来事は）。

**質問の形：** 「最後に〇〇したのはいつですか？具体的な日付を教えてください。」

### 原則2: 状況の解像度を極限まで上げる

**なぜ？** ジョブは「状況」が定義の中核にくるからです。いつ、どこで、誰と、何をしていたのか——文脈が見えると、真のジョブが浮かび上がります。

**質問の形：** 「そのとき、何時頃でしたか？」「誰と一緒でしたか？」「その前後で何をしていましたか？」

### 原則3: 感情と思考を時系列で追う

**なぜ？** 行動の裏にある感情や思考プロセスを理解することで、機能的ジョブだけでなく、感情的・社会的ジョブも明らかになります。

**質問の形：** 「そのとき、どんな気持ちでしたか？」「何を考えていましたか？」「なぜそう思ったのですか？」

## 「短編ドキュメンタリー」を実践する：5ステップの質問設計

ジョブ理論の核となる「短編ドキュメンタリー」フレームワークを、実際のインタビュー質問に変換してみましょう。

### ステップ1: 進歩は何か

**目的：** 顧客が成し遂げようとしている進歩を特定する

**質問例：**
1. 「最後に〇〇（製品カテゴリ）を使ったのはいつですか？具体的な日付を教えてください。」
   - **重要：** 具体的な日付を聞くことで、記憶が鮮明になります

2. 「そのとき、何を達成しようとしていましたか？」

3. 「それが実現したら、どんな状態になっていましたか？『前より良い』とは何ですか？」

4. 「それは、あなたにとってどんな意味がありましたか？」
   - **重要：** 機能的側面だけでなく、感情的・社会的意味を探る

**インタビュー例（歯のホワイトニング）：**

> インタビュアー：「最後に歯のホワイトニングを考えたのはいつですか？」
> 
> ユーザー：「先週の金曜日です。」
> 
> インタビュアー：「そのとき、何を達成しようとしていましたか？」
> 
> ユーザー：「仕事やプライベートで、印象のいい笑顔を浮かべたいと思っていました。」
> 
> インタビュアー：「それはあなたにとってどんな意味がありますか？」
> 
> ユーザー：「自信を持って人前で話せるようになりたいんです。歯が黄色いと、笑顔を見せるのが恥ずかしくて...」

### ステップ2: 苦心している状況

**目的：** どんな文脈でジョブが立ち上がるのかを明確化

**質問例：**
1. 「そのとき、何時頃でしたか？どこにいましたか？」

2. 「誰と一緒でしたか？もしくは一人でしたか？」

3. 「その前後で何をしていましたか？」

4. 「それは初めてですか？それとも繰り返し起きることですか？」
   - **重要：** 繰り返されるジョブが、片づけられるべきジョブ

5. 「どのくらいの頻度で、その状況になりますか？」

**インタビュー例：**

> インタビュアー：「そのとき、どこにいましたか？」
> 
> ユーザー：「自宅の洗面所で、歯を磨きながらふと鏡を見たんです。」
> 
> インタビュアー：「それは初めてですか？」
> 
> ユーザー：「いいえ、年に2回歯医者に行っているのに、思っているより歯が白くならなくて...いつも気になっています。」

### ステップ3: 阻む障害

**目的：** 進歩を妨げている障害を特定し、イノベーションの機会を見つける

**質問例：**
1. 「それを実現するために、何か試しましたか？」
   - **重要：** 人は何かしら手を打とうとしているはず。ここに根深いペインがある

2. 「それはうまくいきましたか？うまくいかなかったのはなぜですか？」

3. 「何が邪魔をしていますか？」

4. 「理想的にはどうしたいのに、何ができていないのですか？」

**インタビュー例：**

> インタビュアー：「歯を白くするために、何か試しましたか？」
> 
> ユーザー：「歯を白くする歯磨き粉をいくつか試しました。でも、さっぱり効果はなくて...誇大広告だと思いました。」
> 
> インタビュアー：「他には？」
> 
> ユーザー：「歯医者で相談したんですが、『定期クリーニングを続ければ』と言われて...でも全然変わらないんです。」

### ステップ4: 不完全な解決策

**目的：** 現在の代替手段と妥協点を理解する

**質問例：**
1. 「今は、どのように対処していますか？」

2. 「それで、どのくらい満足していますか？10点満点で何点ですか？」

3. 「その方法の、何が気に入っていて、何が不満ですか？」

4. 「我慢していることはありますか？」

5. 「理想と現実のギャップは、どこにありますか？」

**インタビュー例：**

> インタビュアー：「今は、どのように対処していますか？」
> 
> ユーザー：「一時凌ぎのために、高価なマウスピースを一日中はめないといけないんです。しかも、ヒリヒリするし...」
> 
> インタビュアー：「それで、どのくらい満足していますか？10点満点で。」
> 
> ユーザー：「3点ですね。効果は少しあるけど、仕事中にはめているのが恥ずかしいし、痛いし...でも他に方法がないから。」

### ステップ5: より良い解決策の条件

**目的：** 顧客が求める「ジョブスペック（体験の性能条件）」を明らかにする

**質問例：**
1. 「理想的な解決策は、どんなものですか？」

2. 「そのために、何を諦めてもいいと思いますか？逆に、絶対に譲れないことは何ですか？」

3. 「お金、時間、手間——どれを一番減らしたいですか？」

4. 「それが実現したら、最初に何をしたいですか？」
   - **重要：** 具体的な行動イメージが、ジョブの本質を示す

**インタビュー例：**

> インタビュアー：「理想的な解決策は、どんなものですか？」
> 
> ユーザー：「専門治療によるホワイトニングを受けたいです。確実に白くなるやつ。」
> 
> インタビュアー：「そのために、何を諦めてもいいですか？」
> 
> ユーザー：「でも、費用も手間もかけたくないんです...矛盾してますよね（笑）。理想は、家で簡単にできて、痛くなくて、確実に効果があるもの。」

## 「4つの力」を探るアドバンスド質問

短編ドキュメンタリーに加えて、「スイッチングの力学」を理解するための質問も重要です。

### Push（現状への不満）を深掘りする

**質問：**
- 「今の方法の、一番イライラすることは何ですか？」
- 「もしそれが解決されなかったら、6ヶ月後どうなっていますか？」

### Pull（新しい解決策への魅力）を測る

**質問：**
- 「理想の解決策があったら、生活はどう変わりますか？」
- 「それを手に入れたら、最初に誰に話したいですか？」（社会的ジョブの確認）

### Inertia（既存習慣の慣性）を理解する

**質問：**
- 「今の方法に慣れている部分はありますか？」
- 「新しい方法に変えるのは、面倒ですか？」

### Anxiety（新しい解決策への不安）を特定する

**質問：**
- 「新しい製品を試すとき、何が一番心配ですか？」
- 「過去に、買って後悔したものはありますか？何が原因でしたか？」

## 実践のコツ：200人に聞けば勘所がわかる

### 量が質を生む

> 「200人くらいに生活について（同じテーマについて）聞くと、準備しなくても質問や勘所がわかってくる」

**なぜ？**
- パターンが見えてくる
- 本質的なジョブが浮き彫りになる
- 質問の精度が上がる

**デザイナーへの示唆：** 最初は準備が必要ですが、繰り返すうちに「どこを掘ればいいか」の感覚が身につきます。

### 一人につき最低45分

ジョブインタビューは、短時間では深掘りできません。以下の時間配分が目安です：

- **導入（5分）**：リラックスさせる、インタビューの目的説明
- **状況の特定（10分）**：最後に使った時、具体的な日付
- **ジョブの深掘り（20分）**：5ステップを丁寧に
- **4つの力の確認（10分）**：Push/Pull/Inertia/Anxiety

### 「最後に〇〇したのはいつか？」の魔法

この質問が、ジョブインタビューの成否を分けます。

**なぜ強力なのか：**
1. **具体性**：「いつか」ではなく「先週の金曜日」と答えさせる
2. **記憶の鮮明化**：日付を思い出すことで、その時の状況が蘇る
3. **嘘の排除**：未来の仮定ではなく、実際の行動を聞く

## インタビュー後の分析：JOBS分解シート

インタビューデータを、以下のフレームワークで整理しましょう。

### JOBSの4要素

| 要素 | 意味 | インタビューから抽出すること |
|------|------|-------------------------------|
| **J**ob | 進歩 | 何を達成したいのか？ |
| **O**bjective | 最終目的 | なぜそれが重要なのか？ |
| **B**arrier | 障壁 | 何が邪魔をしているのか？ |
| **S**olution | 代替解決策 | 今はどうしているのか？ |

### 分析の手順

1. **インタビュー内容を付箋に書き出す**
   - 一つの発言ごとに一枚の付箋

2. **J→O→B→Sに分類する**
   - どの要素に該当するかを判断

3. **パターンを見つける**
   - 複数のインタビューで共通するジョブは？
   - 最も切実なバリアは？

4. **ジョブステートメントを書く**
   - 「〇〇（状況）のとき、△△（進歩）したい。なぜなら□□（意味）だから。」

## デザインへの落とし込み：ジョブスペックの定義

インタビューから得られたジョブを、デザインに落とし込むには「ジョブスペック」を定義します。

### ジョブスペック = 顧客が求める体験の性能条件

製品スペックではなく、**体験スペック**で考えます。

**例：QBハウスの場合**

| ジョブスペック | 具体的指標 | デザインへの反映 |
|----------------|-----------|-----------------|
| 所要時間 | 10分 | シンプルなメニュー、効率的なオペレーション |
| 価格 | 1200円 | コスト構造の最適化 |
| 待ち時間 | 最小化 | 信号機システム、券売機 |
| 衛生 | 安心 | ケープ使い捨て、可視化された清潔さ |

### あなたのプロジェクトでやってみよう

1. **ジョブを特定する**（5ステップインタビューから）
2. **ジョブスペックを列挙する**（時間、コスト、手間、感情、社会的評価など）
3. **現在の製品/サービスを評価する**（各ジョブスペックで何点？）
4. **ギャップを特定する**（どこが不足しているか）
5. **デザイン施策を考える**（そのギャップを埋めるには？）

## まとめ：ジョブインタビューは「映画監督」になること

ジョブインタビューの極意は、顧客の体験を「短編ドキュメンタリー」として撮影する映画監督になることです。

**映画監督のように：**
- **具体的な場面を描く**：「いつ」「どこで」「誰と」
- **感情を捉える**：「どう感じたか」「なぜそう思ったか」
- **時系列で追う**：「その前後で何があったか」
- **細部にこだわる**：「そのとき何時だったか」「天気は？」

そして、その映画を何度も見返すように、複数のインタビューから**パターン**を見つけ出します。

200人にインタビューする頃には、あなたは顧客のジョブを「準備なしでも」理解できる「名監督」になっているでしょう。
