# 観察の技術と哲学：深く見る力の育て方

観察とは、**客観的かつ深く見て、組織化すること**である。単に「見る」ことではなく、見たものを言語化し、構造化し、意味を見出していく営みである。観察の巧拙は、デザイン・リサーチ・ビジネス・創作など、あらゆるジャンルにおける成功の要因となる。

---

## 🎯 観察とは何か

### 観察の定義

> **「観察 = 客観的かつ深く見て、組織化すること」**

観察は以下の3つの要素から構成される：

1. **見る**：対象を注意深く眺める
2. **言語化する**：見たものを言葉にする
3. **組織化する**：言語化したものを構造化し、意味を見出す

### デザインリサーチにおける観察

デザインリサーチでは、観察は**日常にこそ答えがある**という前提に立つ。制御された環境ではなく、実際の文脈の中で人々を観察し、普段の行動に注目する。

**参与観察（Participant Observation）：**
- 調査者自らが現場に入り込み、内側から観察する
- ヤンキー集団の「パシリ」になることで、ネットワークに「埋め込まれる」
- 下にいる方が全体が見れてコミュニケーションが密になる場合もある

## 🔍 観察の方法論とフレームワーク

### AEIOUフレームワーク

エスノグラフィー実践により得られた観察結果を解釈するための経験則的な枠組み（Doblin社、1991年）。

| 要素 | 内容と問いかけ |
|------|---------------|
| **Activities（活動）** | 目的を持った行動やその流れ。「人々は何を達成しようとしているのか？」「どんな手順でそれを行っているのか？」 |
| **Environments（環境）** | 活動が行われる空間全体。「空間の性質や機能は？」「個人・共有スペースの違いは？」 |
| **Interactions（相互作用）** | 人と人、あるいは人とモノの間で起こるやり取り。「どのようなインタラクションがある？」「距離や関係性に特徴はあるか？」 |
| **Objects（物）** | 活動に使用される物理的な道具・デバイス。「何が使われているか？」「その使われ方や意味は？」 |
| **Users（ユーザー）** | 観察の対象となる人々。「誰がいるのか？」「どんな価値観やニーズを持っているか？」 |

**プロセス：**
1. エスノグラフィー的手法を用いて資料を収集（メモ、写真、動画、インタビュー）
2. 現地観察中にAEIOUフレームを使って見る視点を持つ
3. 5つの要素に分類して記録する
4. 複数の観察を統合して、上位のテーマやパターンを抽出する

### 参与観察の技法

**段階的なアクセス：**
1. 知人に紹介してもらう
2. 実習として1日過ごす
3. ボランティアとして住み込み
4. 職員として雇用される
5. 研究倫理を重んじ、改めて調査許可を取る

**位置取りの選択：**
- **下から入る**：全体が見れてコミュニケーションが密になる
- **上から入る**：上位ポジションからでしか見えない層にアクセス

## 🧠 観察の哲学：見ることの本質

### エポケー（判断保留）

> **「わかった気になったら、エポケーと繰り返そう。絶対などない。いい観察は本能とは逆。つまり宙ぶらりんな身を置くこと」**

**エポケー（epoché）** とは、フッサールの現象学における概念で、「判断を留保すること」を意味する。観察においては、すぐに結論を出さず、わからなさに耐え続けることが重要。

**実践：**
- わかった気になっても、「本当にそうか？」と自問する
- 一度グルーピングしても、何度も崩す
- パターンが見えてくるまで粘る

### 問いと仮説の無限ループ

> **「いい観察は問いと仮説の無限ループが起きる。わかった気になってもすぐにまた別のわからないが見つかる」**

**観察のプロセス：**
1. 観察する
2. 問いが生まれる
3. 仮説を立てる
4. また観察する
5. 新しい問いが生まれる
6. ∞ループ

このループこそが、深い理解への道である。

### 環世界（Umwelt）の自覚

> **「自分の見えているこの世界の、全く同じフィールドの背景には、確かに存在しているが見えていない何かがある」**

環世界とは、生物学者ユクスキュルが提唱した概念で、「生物それぞれが知覚し、行動する固有の世界」のこと。

**観察への示唆：**
- 自分の環世界を疑うこと
- いつでも放り出すような身軽さが大事
- 情熱や努力が高まるほど、周囲の流れや現実に気づく力が弱くなることがある

**例：**
- テレビメーカーがいくら大画面・高画質を極めても、社会はスマホで見るNetflixを求めるかもしれない
- 自分の「正しさ」に固執すると、社会の変化が見えなくなる

## 🎨 観察力の鍛え方

### 1. 言語化すること

> **「観察のコツはとにかく言葉にしてみること。必然的に観察時間が伸びる。仮説も思い浮かびやすい」**

**なぜ言語化が重要か：**
- ディスクリプション（記述）する行為が、観察時間を延ばしてくれる
- 言葉にすることで、漠然とした印象が具体化される
- 仮説が立てやすくなる

**実践方法：**
- **事実と主観を分ける**：でないと観察は止まる
- 映画を見て細かく気づきや観察を言語化し仮説を立てる（観察トレーニング）
- 同時に、他人の考察も見て複眼的に見る

### 2. 真似ること

> **「真似ることから全てが始まる。オリジナリティは真似ることなくして生めない。型がなくしてオリジナリティは生めない」**

**真似ることの価値：**
- 真似ようとすると、意外とできなくて、絶望することから始まる
- **観察と仮説と検証のサイクルが真似るということ**
- 型を忠実に守りつつ、その中に何を入れるかが個人のオリジナリティ

**実践：**
- Podcastもnoteもまず真似ることから
- 何を踏襲し、何を踏襲しなかったかを言語化する

### 3. 他ジャンルの観察上手を見つける

> **「観察のうまさがどのジャンルにおいても成功の要因となる。他のジャンルで上手い観察をしている人を見つけるといい」**

**おすすめジャンル：レストラン**
- 再現性を高める工夫
- 領域外を取り入れる工夫
- 環境や社会全体まで含めてデザインする工夫
- 環境ごとデザインする人は、体験そのものが新しいことがある

**他のジャンル：**
- 映画監督
- プロダクトデザイナー
- 建築家
- アーティスト
- 料理人

### 4. 無駄口を叩くこと

> **「無駄口を叩くことがセンスや観察を磨く。雑談や対面が強いのはこれか」**

**なぜ雑談が観察を磨くのか：**
- 無目的な対話の中で、思いがけない気づきが生まれる
- 相手の反応を観察しながら、自分の言葉を調整する
- 「いどばた」（井戸端会議）の価値を再認識

### 5. 反射神経を鍛える

> **「反射神経が必要。面白いところにパクッと飛びつく」**

**観察における反射神経：**
- 観察中に「あれ？」と思った瞬間を逃さない
- 矛盾や違和感に即座に注目する
- 予定していた観察ポイントを一旦捨てる勇気

**鍛え方：**
- 同じテーマで繰り返し観察する（200回くらい繰り返すと勘所が身体化する）
- 何が起きたかだけでなく「自分がどう感じたか」も即座に記録
- アドリブ形式の観察トレーニングを週1回実施

### 6. 平均ラインを内蔵する

> **「自分の中に、客観視できる平均ラインを備えて観察する」**

**平均ラインの重要性：**
- 平均的人物・常識人だからこそ、環境の違和感や特徴を掴める
- 予想外の出来事が起きても「自分のせい」と矮小化せず、環境の特徴として捉える
- 自分の中に確かな尺度がベースとして残っていると、それとの差分が面白みになる

### 7. 全身で浸かりつつアイロニーを保つ

> **「どっぷり参与しながらも、どこか一歩引いて違和感を検知する『遊び』と余裕が不可欠」**

**バランス：**
- **全身で入り込む**：体で体験し、感情も動かす
- **アイロニー視点**：客観視する余裕を保つ
- でないと現前の違和感を特定できない

## ⚠️ 観察の落とし穴

### 1. 認知バイアスとメガネの問題

> **「人はメガネを絶対に外せない。むしろそれを利用し強みに変えることを考えるべき」**

**バイアスの両面性：**
- **プラス面**：直感的な自己防衛としては役に立つ
- **マイナス面**：深く静かな思考としてはむしろ邪魔

**対応策：**
- メガネを外そうとするのではなく、「自分はどんなメガネをかけているのか」を自覚する
- 違和感や不協和こそが、バイアスを超えるトリガーになる
- 複数の視点（他者の考察、チームレビュー）を取り入れる

### 2. 早すぎる収束

**問題：**
- まだ十分にデータを見ていないのに結論を出す
- 最初の仮説に固執しすぎる

**対策：**
- エポケーを実践（判断保留）
- 「これは演繹的に導かれているか？」と常に自問
- チームでレビューし、論理の飛躍を指摘し合う

### 3. 主観の押し付け

**問題：**
- 「私はこう思う」で押し切る
- データが示していないことを語る
- 調査前から持っていた「ものの捉え方」を投影しているだけ

**対策：**
- **演繹的に導く**：データから論理的に導かれる結論を
- 複数の事実が同じ方向を指しているかチェック
- 根拠を明示できるか確認

### 4. デジタル化の罠

> **「事実ベースの観察や言語化はデジタル的？実際の現象はアナログなのに」**

**問題：**
- 観察や言語化は、どうしても離散的（デジタル的）になる
- しかし実際の現象は連続的（アナログ的）である
- 言語化した瞬間に、失われるものがある

**対応：**
- 言語化できないものがあることを自覚する
- 写真、動画、スケッチなど、複数のメディアで記録
- 「言葉にならない感覚」もメモしておく

## 🛠️ 実践的な観察のコツ

### 初期の違和感を大切にする

**重要な原則：**
- フィールドに行きはじめた初期のころに抱く素朴な疑問や違和感は、のちに研究の方向性を大きく決める
- 全体像を掴めていない時は、なんでもメモする
- 慣れてくると「当たり前」になってしまうので、最初の違和感を記録する

### 作業を共にする

**参与観察の基本：**
- 「お手伝いさせてください」と言って、一緒に作業する
- 連帯感が生まれ、雑談の中で日常に近い雰囲気で話を聞ける
- 作業しながらの観察は、インタビューでは得られない情報が得られる

### 何度も足を運ぶ

**長期関与の価値：**
- 素人であっても時間をかけただけ、より深みのあるものになる
- 下手でも、何度も観察・インタビューした方がいい示唆が出る
- 年単位で関わること自体が、倫理性の担保となる

### 矛盾に注目する

> **「語りが矛盾していることにこそ、面白い何かが潜んでいる」**

**実践：**
- 「Aさんはこう言っているが、別の場面ではこう言っていた」
- 「言っていることと、実際の行動が違う」
- 矛盾は、その人の複雑さや葛藤を示している

### インタビュー中の非言語観察

**言葉以外を観察する：**
- インタビュイーが一瞬止まる瞬間
- 表情の変化、声のトーン
- 姿勢や視線の動き
- 質問に対する反応のスピード

**フラットかつ観察的な態度：**
- ドライでも淡々と聴けていれば安心感は生まれる
- 相手が悪いところも率直に指摘できる空気を作る
- フラットな態度が、逆に深い観察を可能にする

**記録の工夫：**
- Miroなどのデジタルツールで発言を可視化すると、パターンが見えやすい
- 録音は必須、メモは最小限に（観察に集中するため）
- 事実と感じたこと（主観）を分けて記録

## 📊 観察データの整理と分析

### シンセシス（統合）の2つの切り口

**1. 人基軸（Person-based）**
- ペルソナごとに情報を整理
- 「Aさんはこう考えている」「Bさんはこうだった」
- 個別の文脈を深く理解する

**2. 示唆基軸（Insight-based）**
- テーマや示唆ごとに情報を整理
- 「○○というニーズがある」「△△という課題が共通している」
- パターンや共通性を見出す

**使い分け：**
- ペルソナ設計が目的 → 人基軸
- サービスコンセプト策定が目的 → 示唆基軸
- 両方を行き来することも重要

### シンセシスの原則：主観ではなく演繹

> **「シンセで重要なのは僕の主観ではなく、客観的で根拠のある演繹」**

**プロセス：**
1. **全体像を把握する**：すべての観察記録、メモを俯瞰
2. **グルーピングする**：似たテーマごとに情報を集める
3. **演繹的に示唆を導く**：「なぜこのグループができたのか？」
4. **レビューで検証する**：チームメンバーからの視点を取り入れる

## 🔄 観察と共感のサイクル

### 開発サイクルと共感サイクルの併走

> **「開発サイクルにエネルギーを注ぐのが常。だがより多くの理解をもたらし助けてくれるのは共感のサイクルの方」**

**共感のサイクル（Empathy Cycle）：**
1. **Develop Empathy（共感の獲得）**：観察・傾聴で「内的思考パターン」を獲得
2. **Apply Empathy（共感の適用）**：その知識を元に意思決定・設計・対話を行う

**観察の位置づけ：**
- 観察は共感のサイクルの起点となる
- 一度の観察で終わらず、サイクルを回し続ける
- 観察→理解→適用→観察の無限ループ

**実践：**
- 開発サイクルとは別に、共感のサイクルを意識的に回す
- 観察で得た理解を、次の観察に活かす
- 人間側の学び→製品側の学びを往復する

## 🎓 観察とインサイト

### N=1, 2からでもインサイトは抽出できる

> **「インタビューや観察を通して得られた何気ない行動に対して、『○○だからなのかもしれない』と踏み込んだ想像や考察をすること」**

**インサイトの3条件：**
1. **文章を読んで状況を理解できること**
   - リサーチに関わってこなかったステークホルダーが理解できるレベル
2. **イノベーションのヒントとなっていること**
   - 行動を起こせるものである必要がある
3. **新規性があり、容易に予測できないこと**
   - 既知の情報をインサイトと提示しても意味がない

### 観察からインサイトへのプロセス

1. **観察**：何気ない行動を記録する
2. **言語化**：その行動を言葉で記述する
3. **問いを立てる**：「なぜその行動をしたのか？」
4. **仮説を考察する**：「○○だからなのかもしれない」
5. **一貫性を見出す**：1つ1つの行動が一貫性を持って抽象化される時、それはインサイトとなる

## 🌟 観察の倫理

### 他者の合理性を理解する

> **「社会学、特に質的調査の、もっとも重要な目的は、私たちとは縁のない人びとの、『一見すると』不合理な行為の背後にある『他者の合理性』を、誰にもわかるかたちで記述し、説明し、解釈すること」**

**観察の姿勢：**
- 尊敬・敬意・尊重が出発点
- 「非合理」に見える行為にもその人の合理性がある
- 安易に「理解した」と言うのは暴力である

### 安易な理解は暴力である

> **「他者を軽々しく理解しようとするのは、暴力です」**

**しかし：**
- 感情や経験そのものを理解することはできなくても
- もっと淡々と、冷静に、「世俗的」に、その合理性を記述することはできるかもしれない
- それによって「隣人」になれるかもしれない

### 隣人になること

> **「質的調査に立脚する社会学の究極の目標は、他者の合理性の理解を通じて、私たちが互いに隣人になること」**

観察とは、他者を「わかる」ことではなく、「隣人になる」こと。黙ってそばにいることができるようになること。

## 💪 観察の習慣化

### 日常での観察トレーニング

**映画・ドラマ：**
- 細かく気づきや観察を言語化し仮説を立てる
- 他人の考察も見て複眼的に見る
- キャラクターの行動の背後にある動機を考える

**店舗・レストラン：**
- なぜこのレイアウトなのか
- スタッフの動線はどうなっているか
- 客はどこで立ち止まるか、どこを素通りするか

**公共空間：**
- 駅、公園、図書館などでの人々の行動
- 空間の使われ方、使われていない部分
- 時間帯による変化

### 観察ノートの作成

**基本フォーマット：**
1. **日時・場所・天候・コンテクスト**
2. **観察したこと（事実）**
   - 何が起きたか
   - 誰が何をしたか
   - どのような行動が見られたか
3. **感じたこと（主観）**
   - 自分の印象・感情
   - 違和感や引っかかり
   - 「気分」として掴んだこと
4. **問い**：なぜ？どうして？どのように？
5. **仮説**：〜だからかもしれない
6. **次の観察ポイント**

**AEIOUフレームでの記録：**
- **Activities（活動）**：何をしていたか
- **Environments（環境）**：どこで、どんな空間で
- **Interactions（相互作用）**：誰と、何と関わっていたか
- **Objects（物）**：何を使っていたか
- **Users（ユーザー）**：誰が、どんな人が

**記録のコツ：**
- 初めのうちは何でも記録する（全体像を掴めていない時）
- 初期の素朴な疑問や違和感は特に丁寧に記録（後の研究の方向性を決める）
- 写真、動画、スケッチなど複数のメディアを併用
- その場で書けなくても、即座にデブリーフィング（振り返り）して記録

## 📈 観察スキルの成長段階

### レベル1：見る
- 対象を眺めることができる
- 事実を記録できる

### レベル2：気づく
- 違和感や特徴に気づける
- 「あれ？」と思える

### レベル3：言語化する
- 見たものを言葉にできる
- 事実と主観を分けられる

### レベル4：構造化する
