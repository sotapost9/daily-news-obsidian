# スキル習得の階梯：できるから洞察へ

[[2025-10-15]]の日記より

> 「sd的なムーブがほぼ言われずともできるようになってきた。でも**どうすればここでより深い洞察を得られるだろう**。丁寧に前後の情報を繋ぐのはベースだが。」

スキルを習得するとは、どういうことか？そして、「できる」ようになった後、その先には何があるのか？

---

## 🪜 スキル習得の5段階

### 1. 無意識的無能（知らないし、できない）

**状態：**
- そのスキルの存在すら知らない
- 何ができないかもわからない

**例：**
- サービスデザインという言葉を知らない段階

### 2. 意識的無能（知っているけど、できない）

**状態：**
- スキルの存在は知っている
- 自分ができないことを自覚している
- やり方を学び始める

**例：**
- 「カスタマージャーニーマップを描くべきだ」と知っているが、どう描けば良いかわからない
- インタビューのやり方を本で読んだが、実際にやるとうまくいかない

### 3. 意識的有能（意識すればできる）

**状態：**
- 手順を意識的に追えばできる
- まだ自然にはできない
- マニュアルを見ながら実行

**例：**
- インタビューガイドを見ながら質問ができる
- ジャーニーマップのテンプレートに沿って作成できる

### 4. 無意識的有能（自然にできる）

**状態：**
- 考えなくてもできる
- 習慣化されている
- 「言われずともできる」

**例：**
> 「sd的なムーブがほぼ言われずともできるようになってきた」

- リサーチ計画を自然と立てられる
- インタビュー中に適切な深掘りができる
- ジャーニーマップを素早く描ける

**ここで多くの人は止まってしまう。**

### 5. 意識的マスタリー（メタ認知と洞察）

**状態：**
- できるだけでなく、**なぜそれをするのか**を理解している
- **どうすればより良くなるか**を考えられる
- 状況に応じて応用できる
- 他者に教えられる

**例：**
> 「どうすればここでより深い洞察を得られるだろう」

- なぜこのインタビューで深い洞察が得られたのか、得られなかったのかを分析できる
- 同じ手法でも、状況によって使い分けられる
- 新しい手法を生み出せる

**これが「できる」から「洞察を得る」への転換点。**

---

## 🤔 なぜ「できる」で止まってしまうのか

### 1. 満足してしまう

「できるようになった！」という達成感で満足してしまう。次のステージがあることに気づかない。

### 2. 忙しさに追われる

> 「丁寧に前後の情報を繋ぐのはベースだが」

できるようになると、依頼が増える。納品に追われて、「なぜ？」を考える余裕がなくなる。

### 3. メタ認知の欠如

自分がやっていることを、一段高い視点から見ることができない。

**メタ認知とは：**
- 自分の思考プロセスを観察する
- 「今、私は何をしているのか？」
- 「なぜこの判断をしたのか？」

### 4. フィードバックの不足

「できている」から誰も指摘してくれない。改善の機会がない。

## 🔍 「できる」から「洞察」へ：5つの実践

### 1. 振り返りの習慣化

**やり方：**
- プロジェクトが終わったら、必ず振り返る
- 「うまくいったこと」「うまくいかなかったこと」を書き出す
- **「なぜそうなったのか？」を3回繰り返す**

**例：**
```
インタビューで深い洞察が得られた
↓ なぜ？
インタビュイーが本音を話してくれたから
↓ なぜ？
信頼関係が構築できたから
↓ なぜ？
最初の10分でアイスブレイクに時間をかけたから
```

### 2. 成功パターンと失敗パターンの言語化

[[2025-10-16]]：シンセシスの技法

> 「シンセで重要なのは僕の主観ではなく、客観的で根拠のある演繹」

**やり方：**
- 自分の過去のプロジェクトを並べる
- うまくいったものの共通点を探す
- うまくいかなかったものの共通点を探す
- パターンを言語化する

**例：**
- 「クライアントとの初期ミーティングで、目的を明確化できたプロジェクトは成功率が高い」
- 「リサーチ対象者の選定に時間をかけたプロジェクトは、洞察の質が高い」

### 3. 他者の実践を観察する

**やり方：**
- 同僚や先輩のやり方を観察する
- 「あの人はなぜそうしたのだろう？」と問う
- 自分のやり方との違いを分析する

**例：**
[[2025-10-16]]の気づき：
- 「S1人でデザインすごい」
- 「あんぬさんもテック橋渡しすげえ」

→ 何がすごいのか？どういうスキルなのか？を分解する

### 4. 意図的な実験

**やり方：**
- 「今回はこれを試してみよう」と意識的に設定する
- 結果を観察する
- うまくいったら、なぜうまくいったかを分析する
- うまくいかなかったら、何が足りなかったかを分析する

**例：**
- 「今回のインタビューでは、最初に必ず『最近どうですか？』と雑談から入ってみよう」
- 「今回のジャーニーマップでは、感情の起伏をより細かく記録してみよう」

### 5. 教えることで学ぶ

**やり方：**
- 後輩にやり方を教える
- ブログや記事に書く
- 社内勉強会で発表する

**教えると何が起きるか：**
- 自分の知識の穴に気づく
- 暗黙知を言語化する必要に迫られる
- 質問に答えることで、自分の理解が深まる

## 🌱 成長の非連続性

### プラトー（高原現象）

スキル習得の過程では、**プラトー**という停滞期がある。

```
成長曲線

│     ┌────────── 意識的マスタリー
│     │
│   ┌─┘←プラトー（停滞期）
│   │
│ ┌─┘
│ │
└─┴──────────────────
```

**プラトーの特徴：**
- 「できるようになった」と感じた後、成長が止まる
- 同じことを繰り返しているだけ
- 新しい挑戦をしていない

**プラトーを突破するには：**
1. 現状に満足しない
2. より高い基準を設定する
3. 新しい文脈で試す
4. フィードバックを求める

### 意志と主体性

[[不幸は楽。幸福は意志と主体性]]

> 「不幸に甘んじるって実はすこぶる快感かつ楽なことで、幸福を目指して生きていくって想像よりもずっと勇気や体力がいること」

**「できる」に甘んじるのは楽だ。**
- 慣れた仕事を繰り返す
- 新しい挑戦をしない
- 失敗を恐れる

**「より深い洞察」を求めるには、意志と主体性が必要だ。**
- 現状に満足しない勇気
- 失敗を恐れずに実験する覚悟
- 学び続ける体力
