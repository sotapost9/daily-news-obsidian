## 2. サービスデザインとビジネスの融合
サービスデザインの視点から、最新のビジネス動向を分析します。

| カテゴリ | 内容 | サービスデザイナーの視点 |
| :--- | :--- | :--- |
| **AIとデザイン** | GenUI（生成型UI）とVibe Coding（ユーザーがAIに構築を依頼する）の対比。 | デザイナーの役割は「画面設計」から「AIの振る舞いや生成ロジックの設計」へとシフトする。 |
| **ビジネスモデル** | Outcome-Oriented Design（成果志向のデザイン）。平均的なユーザーニーズではなく、個別の目標達成を最適化。 | サービス提供側が「何を提供するか」ではなく、ユーザーが「何を実現したいか」に基づいた動向的なサービス設計。 |
| **消費者トレンド** | 『繕いアップデート』：古くなったモノを捨てるのではなく、修復やアップデートを通じて愛着を深める習慣。 | 循環型経済（サーキュラーエコノミー）を前提とした、長期的な顧客関係（LTV）を構築するサービス設計。 |
| **組織とCX** | 建設業界のようなシステム駆動型組織でのCX文化の醸成。 | サービスデザインを単なるツールとしてではなく、組織文化の変革（EXからCXへの接続）として捉える。 |

## 3. サービスデザイナーのインサイト
### デザインプロセスの「圧縮」と「成果志向」
AIの普及により、従来の長大なデザインプロセスは「圧縮」され、より「成果（Outcome）」にフォーカスするようになります。デザイナーは「どのように作るか」よりも「なぜそれを作るのか」「どのような価値を生むのか」という戦略的思考がより重要になります。

### 「繕い」に見る新しい顧客体験
博報堂の「繕いアップデート」は、サステナビリティが個人の楽しみや習慣として定着しつつあることを示しています。サービスデザイナーとしては、製品の「終わり」をデザインするのではなく、修復やカスタマイズを通じて「愛着が育つプロセス」をサービスとして提供する機会があります。

### AIとの共創：GenUIとVibe Coding
ユーザー自身がAIを使ってプロダクトを構築する「Vibe Coding」の時代において、プロのデザイナーが提供すべきは「優れた完成品」だけではなく、「ユーザーが望むものを正しく生成できるためのフレームワーク」です。これはサービスデザインにおける「コ・クリエーション（共創）」の究極の形と言えるかもしれません。
